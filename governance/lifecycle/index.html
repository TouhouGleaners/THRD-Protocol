    维护者生命周期：退出、休眠与除名

    维护者身份的解除分为以下三种情况。

    1. 主动退出 (Voluntary Resignation)

    • 流程：维护者因精力或个人原因拟退出，需发起一个 PR，将其在 access-control.md 中的状态修改为 Emeritus (荣誉成员)。
    • 处理：合并后，相关业务仓库的写权限将予以收回，但其历史贡献将永远保留在 About 页与 Git 记录中。

    2. 自动休眠 (Dormancy)

    为了平衡“资产安全”与“成员归属感”，THRD 实行灵活的休眠保护机制。

    • 休眠定义：连续超过 180 天 未在 GitHub 组织内留下任何协作轨迹（Commit/Issue/Comment）。
    • 休眠处理：相关账号的 业务代码仓库（如 TMA, Danmaku Sender）写权限 将暂时关闭，仅保留 基础文档（Protocol）的维护权限。
      • 目的：这仅仅是为了在成员长期不在岗时，降低账号被盗导致的生产环境风险。
    • 一键复苏：休眠成员回归后，无需重新考核。只需提交一个 PR 或在 Issue 留言说明，即可由核心维护者手动恢复原有权限。

    3. 强制除名 (Removal for Cause)

    维护者的严重违规行为经以下流程处理：

    1. 降权与归档：任一核心维护者确认违规事实后，立即收回所有写权限，并在群聊中公告暂停其维护者身份。24 小时内在 THRD-Protocol 开 Issue 说明情况并归档。
    2. 裁决：归档后 7 天内，由核心维护者在归档 Issue 中发起表决（明确表态“同意除名”或“反对除名”），需多数通过。
      • 表决通过 → 除名生效。
      • 表决未通过 → 恢复原权限。
    3. 申诉：除名生效后 7 天内，被除名者可发起申诉 Issue，核心维护者团队需在 7 天内完成复核，多数通过则维持除名，否则恢复权限。逾期不再受理同一事件的申诉。

    所有除名操作必须在 THRD-Protocol 中开 Issue 归档记录。
