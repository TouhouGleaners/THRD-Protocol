# 沟通规范与异步协作 (Communication Norms)

在 **THRD | 拾遗梦茶馆**，沟通是为了**推动共识达成**。我们通过明确的分工与异步机制，确保组织在低压力环境下高效运行。

---

## 1. 平台定位与信息流转

我们严格区分“即时交流”与“正式记录”：

1.  **即时通讯 (QQ 群)**：
    - **作用**：流转信息、紧急呼叫、日常吹水。
    - **特征**：信息是**瞬时**的。可以用来交换意见，但不作为最终的执行依据。
2.  **GitHub (Issue/PR)**：
    - **作用**：决策存档、方案评审、制度修订。
    - **特征**：信息是**永久**的。是茶馆运行的唯一**正式共识依据**。

---

## 2. 问题路由：决定去哪里说话

为了保持讨论的聚焦，维护者需根据事项性质选择正确的仓库：

- **具体项目事务**（如 TMA 抓不到视频、发射器报错）：
    - **直奔对应项目的仓库**提交 Issue。
- **组织级事务**（如管理制度改革、新项目立项、跨项目协调、重大活动）：
    - **在 THRD-Protocol 仓库**发起讨论或提交 PR。

---

## 3. 协作底线

为了维护协作的专业性与效率，请遵守以下底线：

1.  **严禁手机翻拍屏幕**：所有技术反馈、截图展示必须使用系统原生截图。保持信息清晰是维护者的基本素养。
2.  **未记录即未达成**：涉及权限分配、资源变动、制度修改的决策，如果最终没有转化为 Protocol 仓库中的 PR 或 Issue 总结，则视为**尚未达成最终共识**。
3.  **结论先行**：在 GitHub 发起重大提案或长篇说明时，请在开头写明 TL;DR（概要），尊重他人的阅读时间。

---

## 4. 异步共识

- **不强求秒回**：GitHub 上的讨论允许存在 24-48 小时的响应周期。
- **尊重专注力**：避免在群内频繁 @ 全体成员，除非发生核心服务瘫痪等紧急事故。
