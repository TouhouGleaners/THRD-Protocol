# 技术要求与维护者选修路径 (Technical Standards)

在 THRD Protocol 框架下，维护者需在 **Python (后端/工具)** 与 **Vue 3 (前端)** 两个技术方向中任选其一进行掌握。这是为了确保核心项目（如 Touhou Memory Archive、弹幕发射器等）在未来具备多点维护的能力，降低单点故障风险。

## 1. 基础协作规范 (必修)
*所有维护者无论选修方向，均需掌握以下基础工具，用于参与组织决策、日常协作与文档维护。*

- **Git & GitHub**：掌握基础的 `Pull Request` 流程、`Issue` 讨论及版本回溯。
- **Markdown**：掌握标准 Markdown 语法，用于编写 SOP、技术文档及组织公告。

---

## 2. 选修路径 A：后端逻辑与自动化 (Python)
*适用场景：TMA 爬虫逻辑、失传状态监控、弹幕发射器后端维护及各类自动化脚本。*

### **最小准入范围：**
- **基础语法**：变量定义、标准数据类型、基础运算符。
- **流程控制**：分支语句 (`if-else`)、循环语句 (`for`, `while`)。
- **数据容器**：列表 (`List`)、元组 (`Tuple`)、字典 (`Dict`)、集合 (`Set`) 的增删改查。
- **模块化**：函数 (`def`) 的定义与调用、模块 (`import`) 的导入与管理。
- **面向对象 (OOP)**：理解类 (`Class`) 与对象 (`Object`) 的基本概念。

---

## 3. 选修路径 B：前端交互与可视化 (Vue 3)
*适用场景：TMA 监控看板开发、官网功能嵌入。*

### **最小准入范围：**
- **框架基础**：Vue 3 组合式 API (`setup`) 模式。
- **响应式系统**：理解并能使用 `ref` 和 `reactive` 进行状态管理。
- **指令系统**：熟练使用 `v-if` (条件渲染)、`v-for` (列表渲染)、`v-model` (双向绑定)、`v-bind` (属性绑定)。
- **混合开发**：理解如何将 Vue 组件嵌入现有 WordPress 页面，或通过 API 调用数据。

---

## 4. 全栈鼓励机制 (Full-stack)

虽然准入标准为“二选一”，但组织高度鼓励维护者向全栈方向演进：

- **独立闭环**：具备同时处理数据抓取与前端展示的能力，可以独立负责核心项目模块。
- **协作效率**：在跨端协作（如 API 接口定义）时，具备更强的沟通效率。

---

## 5. 官网维护策略 (WordPress)

针对现有的 WordPress 官网，我们采取**灵活维护**原则：

1. **存量维护**：支持继续使用 WordPress 自带编辑器进行手动文章发布与管理。
2. **增量改造**：鼓励通过 Vue 3 编写极简的前端组件嵌入 WP 页面，以改善交互体验并降低对臃肿插件的依赖。

---

## 6. 核心项目说明

- **Touhou Memory Archive (TMA)**：失传视频与弹幕的监视站。需要 Python 维护抓取逻辑，Vue 3 维护监控看板。
- **弹幕发射器 (Danmaku Sender)**：现由 **Miku_oso** 维护。后续维护者需具备 Python 后端处理能力（最小准入范围）+ PySide6。
