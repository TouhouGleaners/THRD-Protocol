# 维护者入馆指南 (Onboarding)

欢迎来到 **THRD | 拾遗梦茶馆**。无论你是新加入的成员，还是正在进行技术转型的老管理，本指南将引导你完成茶馆新体系下的第一次“协作实操”。

!!! info "给非技术维护者的核心提示"
    如果你选择的是 [运营维护路径](../skills/tech-stack.md#21-operational-track)，你**不需要**学习复杂的命令行指令，也不需要安装任何专业软件。
    通过**浏览器网页端**（像逛论坛、发贴一样点点鼠标），你就能参与茶馆 90% 的管理决策。

---

## 第一阶段：身份确认

在开始之前，请确保你已完成以下准备：

1.  **拥有 GitHub 账号**：并已开启两步验证（推荐）。
2.  **加入组织**：确保你已被邀请进入 [TouhouGleaners 组织](https://github.com/TouhouGleaners)。
3.  **自学资源**：如果你完全没听过 GitHub，建议先在 B 站搜索并观看“GitHub 网页版基础操作”相关视频（只需观看 **网页版** 部分，约 15 分钟）。

---

## 第二阶段：首个 PR 挑战（入馆任务）

**目标**：在 [权限矩阵](../governance/access-control.md) 中正式领取你的维护者编号与路径。

### **具体操作步骤 (网页端版)：**

1.  **打开文件**：进入 `docs/governance/access-control.md` 页面。
2.  **开启编辑**：在预览界面右上角，点击小铅笔图标 (`Edit this file`)。
3.  **填入信息**：在文件末尾的表格中，填入你的 GitHub ID 和路径（运营/技术）。
4.  **提交更改**：
    - 点击页面右上角的绿色按钮 **`Commit changes...`**。
    - 在弹出的窗口中：
        - **Commit message**：写 `feat: 维护者 [你的名字] 正式入馆`。
        - **分支设置**：选中第二项 `Create a new branch for this commit and start a pull request`。
        - **分支命名**：将默认的分支名修改为符合规范的格式，例如：**`feat/onboarding-mikuoso`**。
5.  **发起 PR**：点击 **`Propose changes`**。在接下来的页面中，确认无误后点击 **`Create pull request`**。

!!! tip "为什么要做这个？"
    当你点击 `Create pull request` 的那一刻，你就在“改规矩”了。核心维护者（如 Miku_oso）审核通过后，你的名字就会永久出现在茶馆的网页上。

---

## 第三阶段：路径选择与进阶

通过首个 PR 挑战后，请根据你在 [技术大纲](../skills/tech-stack.md) 中选定的路径开始学习：

### **如果你选了 2.1 运营路径：**
- **任务**：熟悉 [文档标准](../skills/docs-standard.md)。
- **目标**：能流畅地在网页端通过讨论达成共识。

### **如果你选了 2.2 技术路径：**
- **任务**：配置本地 Git 命令行环境，克隆本仓库。
- **目标**：按照技能清单开始自学 Python 或 Vue 3 的最小准入部分。

---

## 第四阶段：正式打理茶馆
一旦你的首个 PR 被核心维护者合并，你将拥有以下权限与责任：

- **参与决策**：通过 Issue 表达意见，通过 Review 审核他人的 PR。
- **自主维护**：发现文档有误或逻辑漏洞时，随时按上述流程发起 PR。
- **跨界演进**：我们鼓励你在熟悉运营后，尝试向技术路径跨界。
