# 管理层角色定义：三位一体架构

在 **THRD | 拾遗梦茶馆** 中，我们认为管理是服务于创作与开发的支撑力量。  
我们不设立脱离实际工作的行政职级，所有的管理者首先是 **Maintainer（维护者）**。  
大家因为共同的愿景聚集在一起，通过协作确保那些珍贵的梦境（数据与记忆）得以妥善保存。

## 1. 核心理念：共同维护 (Shared Maintenance)

我们推崇“管理即代码”，但这并不意味着冷冰冰的规则，而是一种**清晰、透明的协作方式**：

- **以异步为主**：尊重每个人的时间，重要的事情优先通过 Issue 和 PR 记录，方便回溯，不强求即时响应。  
- **文档先行**：好的文档能减少复读，让新成员更容易上手。  
- **极简主义**：我们追求解决问题的最快路径，拒绝不必要的流程。

---

## 2. 三大角色分工

为了让组织更有序，我们根据成员的兴趣和专长，将维护工作分为三个侧重点：

### A. 技术型 Maintainer (Technical Maintainer)
*——“技术探索与质量支持”*

- **我们期待你**：
    - 参与技术选型与架构讨论（如 Python 生态的应用）。
    - 协助进行代码评审（Code Review），给其他成员提供建设性的建议。
    - 维护 CI/CD 等自动化工具，让大家的开发体验更轻松。
- **侧重点**：关注代码的健壮性与前瞻性，是大家在技术上的依靠。

### B. 行政型 Maintainer (Administrative Maintainer)
*——“组织运行与成员服务”*

- **我们期待你**：
    - **社区引导**：帮助新成员熟悉环境（Onboarding），更新行动指南。
    - **资源协调**：管理组织内的权限、API key 或公共账号。
    - **文档维护**：保持 Wiki 和文档目录的整洁，让信息易于查找。
- **侧重点**：关注组织的温度与效率，是确保大家协作无碍的润滑剂。

### C. 复合型 Maintainer (Composite Maintainer / 领头人)
*——“愿景指引与综合协调”*

- **我们期待你**：
    - **方向引导**：确保各项目的开发不偏离 THRD 的核心愿景。
    - **跨界桥梁**：连接技术与行政逻辑，解决协作中出现的模糊地带。
    - **文化建设**：传播“数字保存”的初心，营造自由且专注的创作氛围。
- **侧重点**：关注长期发展，是组织精神的守护者。

---

## 3. 协作原则

我们不要求你时刻待命，但希望在协作中遵循以下默契：

1.  **尊重记录**：尽量在 Git 历史中留下你的思考过程，这不仅是给别人看，也是给未来的自己看。
2.  **包容与纠错**：每个人都会写出 Bug，PR 的意义在于互相帮助检查，而不是互相指责。
3.  **按需贡献**：根据自己的精力认领 Issue，不需要有心理负担。

---

## 4. 身份的获取与流转

- **加入**：只要你愿意长期参与维护工作，并认同我们的文化，欢迎通过修改 `access-control.md` 或直接与现有 Maintainer 沟通来申请角色。
- **成长**：角色不是固定的，你可以根据兴趣在技术与行政之间灵活转换。
- **休息**：如果近期学业或工作繁忙，只需打个招呼。我们随时欢迎老朋友在空闲时回归。
