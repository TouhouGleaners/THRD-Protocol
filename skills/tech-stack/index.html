    技术要求与维护者技能大纲

    在 THRD Protocol 框架下，技能要求根据维护者的 职能侧重 进行阶梯式划分。
    我们的目标是建立基本的逻辑共识，而非强制全员程序员化。

    1. 核心基础 (全员必修)

    进入维护者行列的基础工具，用于参与组织决策与日常协作。

    • GitHub 网页端协作：
      • 能够在网页上提交 Issue 讨论并使用 Labels（标签）。
      • 掌握网页端的“在线编辑”功能，能够发起简单的 Pull Request (PR)。
      • 理解基本的 PR 审核逻辑（Approve / Request Changes）。
    • Markdown 基础：
      • 掌握标题、列表、加粗、链接、图片的基础语法。
      • 能够编写符合 文档标准 的文案。

    2. 维护路径分工 (Maintenance Tracks)

    维护者需根据自己的职能重心，在以下两个路径中达到对应的最小准入标准。

    2.1 运营维护路径 (Operational Track)

    侧重于内容质量、社群维护与对外发布。

    • 内容审计意识：理解补档视频/弹幕的质量标准。
    • 文档维护进阶：
      • 能够通过 GitHub 网页端起草活动策划、公告或维护组织文档（如成员名单）。
      • 具备在 Issue 中清晰描述业务需求与逻辑的能力。

    2.2 技术维护路径 (Technical Track)

    侧重于工具维护与数据监控。要求在掌握 Git 命令行基础后，在以下两条路线中 任选其一 达到最小基础要求。

    • Git 命令行基础：

      • 掌握本地克隆、分支切换、基本提交。理解 Squash and Merge 的合并策略。

    • 路线 A：Python 逻辑基础（适用场景：TMA 爬虫、danmakusender、自动化脚本）：

      • 基础语法：变量定义、标准数据类型、基础运算符。
      • 流程控制：分支语句 (if-else)、循环语句 (for, while)。
      • 数据容器：列表 (List)、元组 (Tuple)、字典 (Dict)、集合 (Set) 的增删改查。
      • 模块化：函数 (def) 的定义与调用、模块 (import) 的导入与管理。
      • 面向对象 (OOP)：理解类 (Class) 与对象 (Object) 的基本概念。

    • 路线 B：Vue 3 前端基础（适用场景：TMA 监控看板、网页版工具）：

      • 框架基础：Vue 3 组合式 API (setup) 模式。
      • 响应式系统：理解并能使用 ref 和 reactive 进行状态管理。
      • 指令系统：熟练使用 v-if (条件渲染)、v-for (列表渲染)、v-model (双向绑定)、v-bind (属性绑定)。
      • 混合开发：理解如何将 Vue 组件嵌入现有 WordPress 页面，或通过 API 调用数据。

    3. 复合型维护者 (Composite)

    组织的架构中枢，鼓励向全栈演进。

    • 双向能力：同时具备运营路径的敏感度与技术路径（Python 或 Vue）的基础支撑。
    • 逻辑转化：能将运营需求转化为基本的技术逻辑描述，消除沟通断层。
    • 全栈最低标准：掌握 Python 或 Vue 3 其中一条路线的基础语法（变量、流程控制、函数），能看懂简单脚本并进行修改。不要求同时掌握两条路线，但鼓励向另一条路线拓展。

    4. 学习建议

    低压准入说明

    • 非技术维护者：只需掌握 GitHub 网页操作。这是一个下午即可完成的学习量。
    • 技术准入者：我们不要求你立刻就能独立写出复杂的项目，只要你能理解基础语法，具备在他人指导下修改简单逻辑的能力即可。
